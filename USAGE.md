# 对话故事编辑器 - 使用说明

## 概述

这是一个基于Vue3的Cocos Creator扩展插件，用于创建和管理对话故事。该插件提供了可视化的故事编辑器，支持流程图式的剧情编辑、节点连线和条件判断。

## 功能特性

### 1. 三种操作模式

#### 选择模式
- 显示故事库中的所有故事
- 可以选择故事进入预览模式
- 可以新建故事

#### 编辑模式
- **左侧目录面板**：
  - 保存故事
  - 导出故事
  - 退出编辑
  - 新建/编辑故事简介
  - 新建/编辑角色信息
  - 新建/编辑变量

- **右侧故事编辑面板**：
  - 可视化流程图编辑
  - 支持多种节点类型
  - 节点之间通过连线连接
  - 连线中可以包含变量条件判断

#### 预览模式
- 按流程图播放故事
- 显示节点内容和选项
- 执行动作和条件判断

### 2. 节点类型

- **开头节点**：故事的起始点
- **对话节点**：显示角色对话内容
- **动作节点**：执行变量设置等操作
- **过渡节点**：连接不同的剧情分支
- **结束节点**：故事的结束点

### 3. 连线和条件判断

- 节点之间通过连线连接
- 连线可以设置条件判断
- 条件基于左侧目录中的变量
- 支持多种操作符：==、!=、>、<、>=、<=

### 4. 变量系统

- 支持字符串、数字、布尔类型
- 可以在动作节点中修改变量值
- 条件判断基于变量值

## 使用方法

### 安装和启动

1. 将插件文件夹放入Cocos Creator扩展目录
2. 在Cocos Creator中打开编辑器，扩展会自动加载

### 创建故事

1. 点击菜单栏的"新建故事"或"编辑故事"
2. 系统会进入编辑模式
3. 在左侧面板编辑故事信息、角色和变量
4. 在右侧面板创建和连接剧情节点

### 编辑故事

1. 点击"编辑故事"菜单
2. 在选择界面选择要编辑的故事
3. 进入编辑模式进行修改
4. 修改完成后点击"保存故事"

### 预览故事

1. 点击"打开故事"菜单
2. 在选择界面选择要预览的故事
3. 系统会进入预览模式
4. 按流程图播放故事
5. 选择选项或触发动作来推进剧情

### 操作说明

#### 添加节点
- 点击右侧工具栏的节点按钮
- 新节点会出现在画布上

#### 移动节点
- 鼠标拖拽节点即可移动
- 连线会自动更新

#### 连接节点
- 选中一个节点
- 编辑其过渡选项
- 设置目标节点ID和条件

#### 编辑节点
- 点击节点选中
- 在右侧属性面板编辑内容

#### 删除节点
- 需要手动在代码中删除或扩展功能

## 技术架构

### 文件结构
```
vue3-template/
├── source/
│   ├── main.ts                 # 主进程入口
│   ├── panels/
│   │   └── default/
│   │       └── index.ts        # 默认面板
│   └── types/
│       └── story.ts           # 类型定义
├── static/
│   ├── template/
│   │   ├── default/
│   │   │   └── index.html     # 面板HTML模板
│   │   └── sample-story.json  # 示例故事
│   └── style/
│       └── default/
│           └── index.css      # 面板样式
├── package.json               # 插件配置
├── tsconfig.json             # TypeScript配置
└── USAGE.md                  # 使用说明（本文件）
```

### 技术栈
- **Vue 3**：UI框架
- **TypeScript**：类型安全
- **Cocos Creator API**：编辑器集成
- **fs-extra**：文件操作

## 开发说明

```bash
npm install
npm run build
```

### 调试
1. 修改源代码后运行 `npm run build`
2. 在Cocos Creator中重新加载扩展
3. 打开开发者工具查看控制台输出

## 注意事项

1. **文件路径**：所有资源路径使用相对于项目根目录的路径
2. **变量类型**：确保变量类型与条件判断匹配
3. **节点连接**：确保所有跳转节点ID都存在
4. **起始节点**：必须设置起始节点才能预览故事
5. **保存频率**：建议定期保存故事，避免数据丢失
